Кратко о проекте

Проект разделён на клиент, службу и общее ядро. Это позволяет отдельно держать интерфейс, команды, сеть и безопасность.

OKОсновной акцент — запуск ПК по сети и удобное удалённое управление.

Как работает

Телефон→ Интернет→ Роутер→ ПК

Команда отправляется с мобильного приложения и выполняется на компьютере через службу. Для запуска ПК по сети используется magic packet.

Структура

Android клиент→ TCP→ Служба→ Команда
UI приложение и служба взаимодействуют через Named Pipe, а внешние команды идут через TCP.

Режимы

Локальный

Подходит для одной сети. Обычно используется локальный IP, например 192.168.x.x.

Внешний

Нужны белый IP, проброс UDP/TCP портов и корректная настройка роутера.

Инструкция

1

Настройка ПК (BIOS + Windows)

  • Включите Wake-on-LAN в BIOS/UEFI (параметры Wake on LAN, Power On by PCI-E, также выключите ErP).
  • В Windows: Диспетчер устройств → Сетевой адаптер → Свойства → вкладка «Питание»: разрешить выводу из сна, разрешить только magic packet.
  • Во вкладке «Дополнительно» активируйте Wake on Magic Packet.
2

Настройка роутера (локально)

  • Закрепите за ПК постоянный локальный IP через DHCP Reservation (привязка по MAC).
  • Проброс портов НЕ требуется — всё работает внутри вашей сети.
  • Убедитесь, что мобильное приложение и ПК находятся в одной подсети.
3

Настройка приложения

  • Привяжите по QR ваш ПК.
  • Включите пункт "Использовать локальную сеть"
1

Подготовка: белый IP и WoL на ПК

  • Убедитесь, что у провайдера белый внешний IP. Проверьте через 2ip.ru или через PC Remote — он должен совпадать с IP в настройках роутера.
  • Включите WoL в BIOS и Windows (см. шаг 1 локальной инструкции).
2

Настройка роутера (проброс портов)

  • Закрепите за ПК постоянный локальный IP (DHCP Reservation).
  • Для Wake-on-LAN: проброс UDP порта. Внешний порт 40000 → внутренний IP ПК, внутренний 40000.
  • Для команд PC Remote: проброс TCP порта. Внешний порт 5055 → внутренний IP ПК, внутренний 5055.
  • Сохраните настройки и перезагрузите роутер.
3

DDNS (если IP динамический)

  • Используйте DDNS, чтобы привязать доменное имя к меняющемуся IP.
4

Настройка приложения

  • Привяжите по QR ваш ПК.
  • Если используете DDNS, укажите доменное имя вместо IP.
⚠️ Общие проблемы WoL
  • ПК не включается — WoL не включён в BIOS или отключён в настройках сетевой карты.
  • Не сделан проброс UDP-порта (для внешнего доступа).
  • Серый IP (CGNAT) — в этом случае внешний доступ без туннеля невозможен.
  • Если ПК включается только первые минуты после выключения, используйте broadcast-адрес (например, 192.168.0.255) вместо IP.
💡 Рекомендации
  • Всегда закрепляйте IP за ПК через DHCP резервацию.
  • Используйте проводное подключение (Ethernet) — WoL по Wi-Fi ненадёжен.

Что такое DDNS?

DDNS (Dynamic DNS) — технология, позволяющая привязать постоянное доменное имя к устройству с динамически меняющимся IP-адресом. Особенно полезна, когда провайдер выдаёт часто меняющийся белый IP.

Как настроить DDNS:

  1. Зайдите в настройки роутера, найдите раздел DDNS, узнайте, какие сервисы он поддерживает.
  2. Зарегистрируйтесь на одном из сервисов (No-IP, DuckDNS, DynDNS).
  3. Выберите доменное имя (например, myhome.ddns.net).
  4. Введите данные от сервиса в роутере (логин, пароль, домен).
  5. Если роутер не поддерживает DDNS, установите клиент на постоянно работающий ПК.
  6. После настройки используйте домен вместо IP в приложении.

Популярные сервисы: No-IP, DuckDNS, DynDNS.
